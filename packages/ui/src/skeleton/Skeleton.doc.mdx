# Skeleton

A skeleton gives feedback about loading state of some component.

<Canvas of={stories.Default} />

## Install

```sh
npm install @spark-ui/ui
```

## Anatomy

```tsx
import { Skeleton } from '@spark-ui/ui/skeleton'

export default () => (
  <Skeleton>
    <Accordion.Rectangle />
    <Accordion.Circle />
    <Accordion.Line />
  </Skeleton>
);
```

## Examples

### Sizing

For `<Skeleton.Rectangle />` component you can use `width|height` props to define the size.

<Canvas of={stories.RectangleSize} />

For `<Skeleton.Circle />` component you can use `size` prop to define the size.

<Canvas of={stories.CircleSize} />

For `<Skeleton.Line />` component you can use `lines` prop to define the number of lines, which will
be equivalent to size in this specific usecase. To handle spacing between these lines, use `gap` specific prop.

<Canvas of={stories.LineSize} />

### Animation

Use `isAnimated` prop to display an animation on your `<Skeleton />` content.

<Canvas of={stories.Animation} />

## Advanced Examples

### Composing layouts

For specific needs you may want to build an entire layout using skeleton items (eg. cards, ...). You may then simply add your own solution (such as a specific markup using flexbox or grid CSS modules) in your implementation.

## Accessibility

<A11yReport of="skeleton" />

The `<Skeleton />` itself shouldn't be visible in the accessibility tree. Still, we strongly recommend to use the `label` prop to allow screen readers to give some equivalent feedback to the user.

### Keyboard Interactions

The `<Skeleton />` shouldn't be accessible using keyboard navigation.
